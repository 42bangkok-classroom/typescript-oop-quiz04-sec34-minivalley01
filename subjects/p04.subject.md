# โจทย์ที่ 4: การอนุมัติและสร้างภารกิจใหม่ (Basic POST Request)

**Path:** `POST /missions`

**Body:** JSON object ของภารกิจใหม่

**ผลลัพธ์:** สร้างภารกิจใหม่พร้อม `id` ที่เป็น auto-increment และบันทึกลงไฟล์ `data/missions.json` พร้อม return ข้อมูลภารกิจที่เพิ่งสร้าง

**ไฟล์ที่ต้องแก้ไข (ไฟล์ที่มีอยู่แล้วจากข้อ 1-3):**

- `src/mission/mission.service.ts` (เพิ่ม method `create(body: any)` เพื่อสร้างภารกิจใหม่และบันทึกลงไฟล์)
- `src/mission/mission.controller.ts` (เพิ่ม endpoint `POST /missions` รับค่าด้วย `@Body()`)

**ตัวอย่างการเรียก API:**

```bash
curl -X POST http://localhost:3000/missions \
  -H "Content-Type: application/json" \
  -d '{
    "codename": "OMEGA_PROTO",
    "riskLevel": "HIGH",
    "targetName": "General Z",
    "startDate": "2024-05-01"
  }'
```

```json
{
  "id": "3",
  "codename": "OMEGA_PROTO",
  "status": "ACTIVE",
  "riskLevel": "HIGH",
  "targetName": "General Z",
  "startDate": "2024-05-01",
  "endDate": null
}
```

**หมายเหตุเพิ่มเติม:**

- ข้อนี้เน้นการรับข้อมูลจาก Request Body และการทำงานของ Method `POST` ขั้นพื้นฐานแบบไม่ใช้ `ValidationPipe`
- ข้อมูลใหม่ที่ต้องบันทึกลงไฟล์จะต้องมีโครงสร้างเหมือนกับที่มีอยู่ และเซ็ตค่า default ให้ฟิลด์ที่ขาดหายดังนี้:
  - กำหนด `status` เป็น `"ACTIVE"` โดยอัตโนมัติ
  - กำหนด `endDate` เป็น `null` โดยอัตโนมัติ
  - สร้าง `id` เป็นเลขหมายถัดจากรายการสุดท้ายใน data (เช่น ถ้า id ล่าสุดคือ "2", อันใหม่ต้องเป็น "3")
- ต้องเขียนข้อมูลที่มีการเพิ่มรายการใหม่ลงทับในไฟล์ `data/missions.json`

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `missions.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
