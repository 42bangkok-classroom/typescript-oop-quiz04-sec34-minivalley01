# โจทย์ที่ 1: สรุปสถานะภารกิจลับ (Data Aggregation & Reduction)

**Path:** `GET /missions/summary`

**ผลลัพธ์:** JSON object แสดงผลรวมจำนวนภารกิจแยกตามสถานะ

**ไฟล์ที่ต้องสร้าง:**

- `src/mission/mission.controller.ts` (class `MissionController`)
- `src/mission/mission.service.ts` (class `MissionService`, method `getSummary()`)

**ไฟล์ที่ต้องแก้ไข:**

- `src/app.module.ts` (import และเพิ่ม `MissionController`, `MissionService`)

**เงื่อนไข:**

- ใน `MissionService` ให้จำลอง Array ของภารกิจ (Mock Data) ดังนี้:
  ```typescript
  private readonly missions = [
    { id: 1, codename: 'OPERATION_STORM', status: 'ACTIVE' },
    { id: 2, codename: 'SILENT_SNAKE', status: 'COMPLETED' },
    { id: 3, codename: 'RED_DAWN', status: 'FAILED' },
    { id: 4, codename: 'BLACKOUT', status: 'ACTIVE' },
    { id: 5, codename: 'ECHO_FALLS', status: 'COMPLETED' },
    { id: 6, codename: 'GHOST_RIDER', status: 'COMPLETED' }
  ];
  ```
- Method `getSummary()` จะต้องไม่คืนค่า Array ออกไปตรงๆ แต่ให้ทำการรวบรวม (Aggregate) จำนวนสถานะแต่ละแบบจาก mock data ข้างต้น
- ส่งผลลัพธ์กลับไปเป็น object ที่มี key เป็นชื่อสถานะ และ value เป็นจำนวนนับ

**ตัวอย่างการเรียก API:**

```bash
curl http://localhost:3000/missions/summary
```

**ตัวอย่างผลลัพธ์:**

```json
{
  "ACTIVE": 2,
  "COMPLETED": 3,
  "FAILED": 1
}
```

**หมายเหตุ:**

- ข้อนี้ต้องการทดสอบ Logic การวนลูปหรือใช้ Array helper (เช่น `.reduce()`) เพื่อปรับแต่งข้อมูลก่อนส่งออก

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `missions.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
