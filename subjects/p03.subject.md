# โจทย์ที่ 3: การปกปิดข้อมูลขั้นสูงตามระดับสิทธิ์ (Role-based Data Redaction)

**Path:** `GET /missions/:id`

**Query Parameters:** `clearance` (string) - ระบุระดับสิทธิ์ (เช่น `STANDARD`, `SECRET`, `TOP_SECRET`) (ไม่จำเป็นต้องระบุก็ได้ ค่าเริ่มต้นคือ `STANDARD`)

**ผลลัพธ์:** ข้อมูลภารกิจ 1 รายการ พร้อมการเซ็นเซอร์บางฟิลด์ หากระดับสิทธิ์ไม่ถึง

**ไฟล์ที่ต้องแก้ไขเพิ่มเติม:**

- `src/mission/mission.service.ts` (เพิ่ม method `findOne(id: string, clearance: string)`)
- `src/mission/mission.controller.ts` (เพิ่ม endpoint `GET /missions/:id` พร้อมรับ query `clearance`)

**ข้อมูลอ้างอิง:**

- **ระดับความเสี่ยง (`riskLevel`) ที่เป็นไปได้ทั้งหมด:** `"LOW"`, `"MEDIUM"`, `"HIGH"`, `"CRITICAL"`
- **ระดับสิทธิ์ (`clearance`) ที่เป็นไปได้ทั้งหมด:** `"STANDARD"`, `"SECRET"`, `"TOP_SECRET"`

**เงื่อนไข:**

- อ่านข้อมูลจาก `data/missions.json` เพื่อหาภารกิจที่มี `id` ตรงกัน
- หากหาไม่พบ ให้ throw `NotFoundException` (404)
- **เงื่อนไขเซ็นเซอร์ (Data Redaction):**
  - หากภารกิจนั้นมี `riskLevel` เป็น `"HIGH"` หรือ `"CRITICAL"` แต่ query `clearance` เป็นอย่างอื่นที่ไม่ใช่ `"TOP_SECRET"` ให้เปลี่ยนค่าฟิลด์ `targetName` ใน response เป็น `"***REDACTED***"`

**ตัวอย่างการเรียก API:**

1. ดึงข้อมูลภารกิจทั่วไป (MEDIUM) สิทธิ์ไหนก็ดู target ได้

```bash
curl http://localhost:3000/missions/1?clearance=STANDARD
```

```json
{
  "id": "1",
  "codename": "WINTER_SUN",
  "status": "COMPLETED",
  "targetName": "Dr. Viktor",
  "riskLevel": "MEDIUM"
  ...
}
```

2. ดึงข้อมูลภารกิจอันตราย (HIGH) แต่สิทธิ์แค่ SECRET

```bash
curl "http://localhost:3000/missions/2?clearance=SECRET"
```

```json
{
  "id": "2",
  "codename": "NIGHT_SHADOW",
  "status": "ACTIVE",
  "targetName": "***REDACTED***",
  "riskLevel": "HIGH"
  ...
}
```

3. ดึงข้อมูลภารกิจอันตราย (HIGH) ด้วยสิทธิ์ TOP_SECRET

```bash
curl "http://localhost:3000/missions/2?clearance=TOP_SECRET"
```

```json
{
  "id": "2",
  "codename": "NIGHT_SHADOW",
  "status": "ACTIVE",
  "targetName": "Unknown Syndicate Base",
  "riskLevel": "HIGH"
  ...
}
```

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `missions.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
