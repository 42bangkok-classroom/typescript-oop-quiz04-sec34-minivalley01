# โจทย์ที่ 2: ระบบคำนวณระยะเวลาภารกิจ (Interface & Computed Fields)

**Path:** `GET /missions`

**ผลลัพธ์:** ข้อมูลภารกิจทั้งหมดจากไฟล์ `data/missions.json` พร้อม field พิเศษ `durationDays`

**ไฟล์ที่ต้องสร้าง:**

- `src/mission/mission.interface.ts` (interface `IMission`)
- สร้างไฟล์ `data/missions.json` ที่ root ของโปรเจกต์ (เพื่อนบ้านกับ package.json) โดยมีข้อมูลทั้งหมดดังนี้:
  ```json
  [
    {
      "id": "1",
      "codename": "WINTER_SUN",
      "status": "COMPLETED",
      "targetName": "Dr. Viktor",
      "riskLevel": "MEDIUM",
      "startDate": "2024-01-01",
      "endDate": "2024-01-10"
    },
    {
      "id": "2",
      "codename": "NIGHT_SHADOW",
      "status": "ACTIVE",
      "targetName": "Unknown Syndicate Base",
      "riskLevel": "HIGH",
      "startDate": "2024-02-15",
      "endDate": null
    }
  ]
  ```
  **(หมายเหตุ: ห้ามนักเรียนแก้ไขข้อมูลในไฟล์ `data/missions.json` ด้วยตัวเอง)**

**ไฟล์ที่ต้องแก้ไข (จากข้อ 1):**

- `src/mission/mission.service.ts` (เพิ่ม method `findAll()`)
- `src/mission/mission.controller.ts` (เพิ่ม endpoint `GET /missions`)

**เงื่อนไข:**

- Interface `IMission` ต้องมี Attributes ตามโครงสร้างใน `data/missions.json`
- Method `findAll()` ต้องอ่าน JSON ไฟล์จาก `data/missions.json`
- ก่อนที่จะ Retun ค่า ให้ทำการ Map ข้อมูลเพื่อเพิ่ม field พิเศษชื่อ `durationDays` เข้าไปในแต่ละ object
- `durationDays` (ตัวเลข) ให้คำนวณจำนวนวันจาก `startDate` ไปถึง `endDate`
- ถ้า `endDate` เป็น `null` (ภารกิจยังไม่จบ) ให้ตั้ง `durationDays` เป็น `-1`

**ตัวอย่างการเรียก API:**

```bash
curl http://localhost:3000/missions
```

**ตัวอย่างผลลัพธ์:**

```json
[
  {
    "id": "1",
    "codename": "WINTER_SUN",
    "status": "COMPLETED",
    "startDate": "2024-01-01",
    "endDate": "2024-01-10",
    "durationDays": 9
  },
  {
    "id": "2",
    "codename": "MIDNIGHT_RUN",
    "status": "ACTIVE",
    "startDate": "2024-02-15",
    "endDate": null,
    "durationDays": -1
  }
]
```

**หมายเหตุ:**

- ถือว่าสูตรหาจำนวนวันคือ `(ปลายทาง - ต้นทาง) / เวลาใน 1 วันตามหน่วย milliseconds` หรือใช้ library `Date` ช่วย
- ข้อนี้ทดสอบการกำหนด Interface, อ่านไฟล์ภายนอก, และแปลง (Transform) ข้อมูลก่อนเสิร์ฟ

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `missions.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
