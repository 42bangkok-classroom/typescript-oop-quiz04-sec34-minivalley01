# โจทย์ที่ 5: การยกเลิกภารกิจ (DELETE Request)

**Path:** `DELETE /missions/:id`

**ผลลัพธ์:** ลบภารกิจจากไฟล์ `data/missions.json` พร้อมส่ง API Response กลับมาแจ้งสถานะ

**ไฟล์ที่ต้องสร้าง/แก้ไข (ไฟล์ที่มีอยู่แล้วจากข้อ 1-4):**

- `src/mission/mission.service.ts` (เพิ่ม method `remove(id: string)` เพื่อลบข้อมูลภารกิจ)
- `src/mission/mission.controller.ts` (เพิ่ม endpoint `DELETE /missions/:id`)

**เงื่อนไข:**

- รับ `id` ของภารกิจที่ต้องการลบผ่านทาง URL Parameter (`@Param()`)
- อ่านข้อมูลทั้งหมดจาก `data/missions.json` มาค้นหาว่ามีภารกิจที่มี `id` ตรงกันหรือไม่
- หากหา **ไม่พบ** ให้ throw `NotFoundException` (404 status code)
- หากหา **พบ** ให้ทำการลบภารกิจนั้นออกจาก Array
- เขียนข้อมูลตัวใหม่ที่ถูกลบรายการนั้นออกไปแล้ว ลงทับกลับไปในไฟล์ `data/missions.json`
- รีเทิร์น Object แจ้งเตือนสถานะสำเร็จกลับไป (200 OK)

**ตัวอย่างการเรียก API:**

1. ลบภารกิจที่หาพบสำเร็จ:

```bash
curl -X DELETE http://localhost:3000/missions/1
```

```json
{
  "message": "Mission ID 1 has been successfully deleted."
}
```

2. ลบภารกิจที่ไม่มีอยู่จริง:

```bash
curl -X DELETE http://localhost:3000/missions/999
```

```json
{
  "statusCode": 404,
  "message": "Not Found",
  "error": "Not Found"
}
```

**หมายเหตุเพิ่มเติม:**

- ข้อนี้เน้นการจัดการ HTTP Method `DELETE`
- ต้องการให้ผู้เรียนรู้จักการเขียนแก้ไขไฟล์ภายนอก (Write File) ควบคู่กับการประมวลผล Array รูปแบบพื้นฐาน (เช่น `filter()` หรือ `splice()`)

**ข้อห้าม:**

1. ห้ามใช้ `process.exit()` มิเช่นนั้นจะตัดคะแนนเป็น 0 ทันที
2. ห้ามแก้ไข file `missions.json` หากใช้เพื่อทดสอบเสร็จแล้วให้แก้ไขเป็นข้อมูลเดิมหรืออย่า push ขึ้น git repository.
